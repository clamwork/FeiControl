# FeiControl — 产品迭代路线图

> ✅ 已全部完成 — v1.0.0 已发布
> 本路线图记录了从 v0.1.0 到 v1.0.0 的全部开发阶段。

---

## Phase 1: 基础设施与稳定性（当前 → v0.2.0）

| 优先级 | 任务 | 说明 | 文件/范围 |
|--------|------|------|-----------|
| 🔴 P0 | 添加 Error Boundary | 全局 + 页面级错误捕获，优雅降级 | `src/components/ErrorBoundary.tsx` |
| 🔴 P0 | 添加加载骨架屏 | 每个页面路由的 loading.tsx + Suspense fallback | `src/app/**/loading.tsx` |
| 🔴 P0 | 完善 TypeScript 严格模式 | 修复隐式 any，补充缺失的类型定义 | `tsconfig.json`, 各组件 |
| 🟡 P1 | 添加单元测试框架 | Vitest + React Testing Library | 配置文件 + 首批测试 |
| 🟡 P1 | 添加健康检查自愈 | 定时轮询 API 端点的可用性 | `src/app/api/health/` |
| 🟡 P1 | 环境变量校验 | 启动时校验必需的环境变量 | `src/lib/env.ts` |
| 🟢 P2 | 完善 i18n 文案覆盖 | 补齐缺失的翻译键 | `src/i18n/locales/` |

## Phase 5: 战略选择（v1.0.0 之后）

> 🆕 **v1.0 已发布，下一步是选择产品发展方向，而不是堆功能。**
> 详见 [`docs/PRODUCT_PLAN.md`](docs/PRODUCT_PLAN.md#8-战略发展方向) 第 8 节。

### 四个伴侣方向

| 方向 | 伴侣角色 | 一句话 | 改动 | 定位匹配度 |
|------|----------|--------|:----:|:----------:|
| **A: 情感伴侣** | 朋友 | 你的 Agent 不是一个工具，而是一个有温度的朋友 | 小~中 | ✅ 最高 |
| **B: 效率伴侣** | 管家 | 你负责想，Agent 负责做，FeiControl 负责盯 | 大 | ⚠️ 中 |
| **C: 记忆伴侣** | 档案员 | 你的 Agent 会忘记，但 FeiControl 不会 | 中 | ✅ 高 |
| **D: 守护伴侣** | 安全顾问 | 你的 Agent 在做什么？FeiControl 告诉你 | 中~大 | ⚠️ 中 |

### 当前建议

**A（情感伴侣）> C（记忆伴侣）> D（守护伴侣）> B（效率伴侣）**

### 经验教训（Phase 1-4 复盘）

| 阶段 | 关键教训 |
|------|----------|
| Phase 1: 基础设施 | Error Boundary 和骨架屏是最值的早期投入——让后续开发始终有安全网 |
| Phase 2: 功能增强 | i18n 一开始就做对了，后期补会非常痛苦；通知系统比预期复杂 2x |
| Phase 3: 体验与工程化 | PWA 的离线缓存是用户最喜欢的特性之一，但 Service Worker 调试很痛苦 |
| Phase 4: 生态与高级功能 | 插件系统做了但几乎没人用——功能优先级判断失误 |
