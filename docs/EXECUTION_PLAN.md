### ⚠️ 关键差距

| 维度 | 当前状态 | 目标状态 |
|------|---------|---------|
| 心智模型 | 仪表盘 + 管理工具 | 上帝面板 — 活着的世界 |

| 每次打开 | 跑数据 | **有变化** — 子民完成了、进化了、新降临了 |
| 离线感知 | 无 | 世界没停，回来有惊喜 |
| 期待感 | 改需求/调参数时才打开 | 想看看我的世界怎么样了 |

```
阶段 1 — "这个世界活了"（核心转型）    ← 我们在这里开始
  让 Dashboard 变成活的。不改变代码结构，改变体验。

阶段 3 — "神谕"（交互重塑）
```

## 2. 阶段 1："这个世界活了"（核心转型）

**目标**：用户打开 FeiControl 的第一感受从"我来看看数据"变成"我来看看我的世界怎么样了"。

**不需要改动现有页面的核心功能**。只改变 Dashboard 首页的心智模型、文案、动画、和离线感知。

### 2.1 Dashboard 首页改造

#### 2.1.1 世界问候区（现有问候语改造）

**现在**：`☀️ 早上好 · 3 agents · 1 working · 2 standby`

**改成**：

```
☀️ 早上好，你的世界正在苏醒

━━━━━━━━━━━━━━━━━━━━━━━
3 个子民 · 1 个在忙 · 2 个在休息

📮 昨晚你不在的时候发生了 8 件事
🌱 有一个子民接近进化了（34/100）
━━━━━━━━━━━━━━━━━━━━━━━
```

- 强调"你的世界"，不是"你的系统"
- 把"Agent"改成"子民"（用户可配，默认"子民"）
- 新增"离线发生了什么"的叙事流，而不是数据展示

**具体改动**：
1. 修改 `greeting` 文案系统，加入世界叙事前缀
2. 在问候区增加"离线简报"卡片
3. 文字动画：打开面板时问候语逐字淡入

#### 2.1.2 Agent 状态区 → "住民状态板"

**现在**：Agent 卡片列表，显示 online/offline/busy，CPU 等系统指标。

**改成**：

- 去掉系统健康指标（CPU/内存/磁盘）——这些是运维信息，不是世界信息
- 或者把系统健康移到角落，主视觉给 Agent
- 每个 Agent 卡片增加：
  - **今日干了啥**（1 句简短总结）
  - **状态动词**（"巡游中"、"小憩中"、"在发呆" 替代 "working"/"idle"）
  - **情绪/活力指示**（不是用绿色/灰色，而是用更温暖的表达）
  - **上次你看到它的时候它还没做完的事，现在做完了吗？**

**具体改动**：
1. `Agent 卡片` 增加 `currentTask` 动态状态词
2. 状态显示从技术态（online/offline）改成叙事态（working → "在忙"）
3. 卡片 hover 时显示"今日贡献"摘要

#### 2.1.3 "离线世界" 叙事

**现在**：Cron 任务状态 + 昨日回顾（以数据表格风格呈现）。

**改成**：

「昨晚你的世界中发生了这些事」叙事卡片：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🌙 昨晚（当你睡着时）

• 子民 A 完成了 5 个任务，成长 +5
• 子民 B 自己启动了 1 个任务（你从未安排过）
• 世界共发生 12 次活动，0 次异常

💡 今天你可以考虑给子民 B 下派一个新的神谕
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**具体改动**：
1. `Dashboard` 的 `yesterdayMemory` 组件改名为"离线世界简报"
2. 展示风格从技术表格改为人话叙事
3. 如果昨天没有任何事发生，展示"世界很平静。所有子民都在休息。"

#### 2.1.4 进化感入口

**现在**："Self-Evolution Morning Briefing" 区块。

**改成**：

将进化展示从技术仪表风格改造成"子民成长报"：

- 接近进化的子民高亮展示（发光的卡片/徽章）
- 进度条周围增加叙事文案（"距离下一次蜕变只差 12 次任务"）
- 整体视觉风格从"报告"变成"好消息"

### 2.2 全局文案和术语替换

| 原词 | 替换为 | 场景 |
|------|--------|------|
| Agent | 子民（默认）/ 住民 | 全站 |
| Agent List | 住民列表 / 你的子民 | 标题 |
| Status | 状态 · 在忙/休息/巡游 | 状态指示 |
| Online | 在忙 | Agent 状态 |
| Offline | 静默 | Agent 状态 |
| Busy | 专注中 | Agent 状态 |
| CPU / RAM / Disk | 移至角落或去掉 | Dashboard |
| Tasks | 事迹 / 贡献 | Agent 详情 |
| Dashboard | 你的世界 | 首页标题 |
| System Health | — | 去掉，或缩为一个图标提示 |

> 术语替换优先走 i18n 文案，不改变代码内部变量名。这样保留技术可维护性，只改用户看到的。

### 2.3 首次使用体验改造（"开箱即活的温暖"）

**现在**：用户装好 → 登录 → 看到 Dashboard 数据。

**改成**：

```
用户首次装好 FeiControl:
  → 连接第一个 OpenClaw Agent
  → 面板展示的不是数据，是一段文字：
      "你的世界迎来了第一位住民。它已经坐好了。"
  → Agent 卡片不是 "online"，而是 "在熟悉环境"
  → 界面上没有系统指标，只有这个 Agent 和一句欢迎
```

**具体改动**：
1. 检查 `localStorage` 是否有 `hasSeenOnboarding`
2. 首次登录展示欢迎卡（而非 Dashboard 标准内容）
3. 欢迎卡文案来自 `PRODUCT_PLAN.md` 用户旅程"首次使用"章节

### 2.4 阶段 1 交付标准

```
□ Dashboard 首页不再显示 CPU/RAM/Disk（移至设置或缩略图标）
□ Agent 状态使用叙事化文案（在忙/休息/巡游）
□ "离线简报"替代"昨日回顾"——用完整段落的自然语言展示
□ 问候语强调"你的世界"
□ 接近进化的 Agent 有视觉高亮
□ 全部文案完成术语替换（子民/住民/神谕/世界）
□ 首次使用有欢迎卡
□ 阶段 1 自测：打开页面，读 5 秒，感受到"这是一个世界"而非"这是一个工具"
```

### 2.5 估算工作量

| 任务 | 涉及文件 | 预估工时 |
|------|---------|---------|
| 问候语改造 + 世界叙事前缀 | `page.tsx`（Dashboard）+ i18n 文案 | 2-4h |
| Agent 卡片叙事化（状态词、今日动态） | `page.tsx` + Agent 组件 | 3-6h |
| 离线简报改造（叙事化输出） | `page.tsx` + API 侧格式化 | 4-8h |
| 进化感高亮 | `page.tsx` CSS 动画 | 2-3h |
| 全局文案替换（术语） | i18n JSON 文件 | 2-3h |
| 首次使用欢迎卡 | Dashboard 内部新组件 | 3-5h |
| 系统健康收起/下移 | Dashboard 布局调整 | 1-2h |

**总计：约 17-31 小时（4-8 个工作日）**

## 3. 阶段 2："子民们"（Agent 人格化）

> 让 Agent 从条目变成住民。

### 3.1 Agent 详情页 → "住民档案"

**现在**：Agent 详情展示模型、状态、活动时间线。

**改成**：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🦊 阿远 · 巡游中

出生时间: 2025-06-15
世界身份: 分析师
当前工位: 主办公室 3 号桌

今日贡献:
  ▸ 整理了 5 份项目文档
  ▸ 完成了 1 次数据清洗
  ▸ 当前在: "正在阅读本周的日志…"

成长进度: ████████░░░ 34/100
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**改动**：
1. Agent 详情页增加"住民档案"卡片，展示"出生时间"、"世界身份"、"当前工位"
2. 活动时间线叙事化（"16:32 — 完成了项目文档整理" → "下午 4 点半，阿远完成了项目文档整理，看起来挺满意"）
3. 成长进度可视化增强（不是进度条，是"蜕变道路上"）

### 3.2 Agent 列表 → "住民广场"

**现在**：卡片网格 + 状态指示器。

**改成**：

- 每个卡片更大，展示角色表情和"状态短语"（不是在忙/在线，而是"正在钻研代码"、"刚刚忙完一阵"、"趴在桌上休息"）
- 列表顶部文案从 "3 Agents" 变成 "你的世界里住着 3 位子民"
- 空列表状态从 "暂无 Agent" 变成 "你的世界还空着——连接第一个 Agent，它将是你的第一位住民"

### 3.3 成长线可视化

这是 PRODUCT_PLAN.md 中用户旅程的"进化时刻"。

**现在**：`self-evolution` 区展示技能和其他技术数据。

**改成**：

- 每个 Agent 有显性的成长进度条（不是只有后台知道）
- 接近里程碑时 Dashboard 上有视觉通知（徽章发光、卡片浮动）
- 进化完成时触发庆祝动画（类似游戏升级特效）

### 3.4 阶段 2 交付标准

```
□ Agent 详情页有"住民档案"卡
□ Agent 列表状态是叙事短语
□ 空状态有温暖文案
□ 成长进度条在每个 Agent 卡片上可见
□ 接近进化时有视觉提示
□ 进化完成时有庆祝动画
□ 全部页面完成术语替换
```

## 4. 阶段 3："神谕"（交互重塑）

> 从"发消息"变成"降下旨意"。

### 4.1 神谕面板（替代/增强 Chat 页面）

**现在**：Chat 页面 → 选择 Agent → 输入消息 → 发送 → 收到回复。

**改成**：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🗳 神谕面板

"降下旨意: 你的子民们将收到并自行解读"

┌──────────────────────────────┐
│ [输入你的旨意…]               │
│                              │
│ 示例: "帮我整理本周的笔记"     │
│       "我想知道最近的趋势"     │
│       "去看一下昨天的日志"     │
└──────────────────────────────┘

神谕将送达:
  ☑ 所有子民（或）
  ○ 指定子民: [选择…]

[降下神谕] → "子民已收到你的旨意，正在解读执行方案…"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**改动**：
1. Chat 页面新增"神谕模式"切换（或默认改为神谕面板）
2. 神谕面板不要求指定详细步骤——只输目标
3. 发送后展示"子民收到神谕，开始解读"的动画状态
4. 保留原 Chat 功能作为"神谕详情查看"（查看子民执行过程）

### 4.2 神谕历史

- 所有发过的神谕成为一个时间线
- 每条神谕显示：发出时间 → 哪个/些子民收到 → 执行结果
- 神谕的执行结果不是技术日志，而是叙事总结（"阿远收到你的旨意后，在下午 3 点完成了整理，共整理 12 份文件"）

### 4.3 Notifications → "世界动态"

- 通知条目从技术语言改为人话
- 每条通知看起来像"你的世界的一个消息"，而不是系统告警
- 示例：🦊 **阿远完成了神谕** — "你下午派的任务，阿远已经做好了。共整理了 12 份文件。"
- 通知分组：按"神谕完成"、"子民进化"、"新子民降临"、"日常动态"分类

## 5. 阶段 4："世界自己会生"（自主性）

> 子民不只是你接入的——世界会自己诞生新的住民。

### 5.1 自生 Agent 机制

**思路**：FeiControl 可以检测到用户系统里有没有未被添加的 Agent 实例，自动引入世界。

**具体实现**（无需变更 Agent 运行时）：

1. 启动时/定时扫描文件系统上有没有新的 Agent 数据目录
2. 如果发现新 Agent，在 Dashboard 弹出通知：
   > "你的世界迎来了新的住民。它自己来了。"
3. Agent 卡片展示时添加"降临"标签
4. 自生 Agent 和用户接入的 Agent 在行为上没有区别

### 5.2 实验性：Agent 交叉影响

- Agent A 完成多个任务后，触发 Agent B 开始一个新任务（类似"子民之间的互动"）
- 这不改变 Agent 后端，只是 FeiControl 侧的事件联动

### 5.3 "世界日志"

- 增加一个"世界日志"视图，展示世界层面的叙事：
  - "3 小时前 — 阿远醒来"
  - "1 小时前 — 阿远完成了昨天未完成的任务"
  - "30 分钟前 — 世界孕育了新的住民（林克）"
  - "5 分钟前 — 林克开始整理文件"

## 8. 不做的（对应 ❌ 表）

| 不做 | 原因 |
|------|------|
| 改变现有 Agent 运行时 | FeiControl 只改面板体验，不改 Agent 后端 |
| 移除 Chat 页面 | Chat 被增强/代替，但内核保留（神谕详情查看） |
| WebSocket 升级 | SSE 够用 |
| 多用户系统 | 个人上帝面板，不是团队工具 |
| 云服务 | Local First，所有数据在本地 |
| 工作流编排 | 子民收到神谕后自行决定步骤 |
| 商业模式 | 开源免费 |
